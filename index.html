BACKSTAGE.JS03

Editorial

// una carta antes de empezar

Cuando alguien abre una página web, ve botones, colores y animaciones. Lo que no ve es todo lo que ocurre segundos antes: un servidor que recibe la solicitud, un protocolo que la transporta y un lenguaje que decide qué responder.

Esta edición de BACKSTAGE.JS nació de una pregunta sencilla en clase: si JavaScript ya no es solo "el lenguaje del navegador", ¿qué tan lejos llega cuando se muda al servidor? Las páginas que siguen son nuestro intento de responderla.

— La Redacción

BACKSTAGE.JS05

Introducción

// el modelo cliente-servidor

Toda página web que visitas es, en realidad, una conversación entre dos partes: el cliente (tu navegador) y el servidor (una computadora en algún centro de datos). El cliente pide algo, una imagen, un dato, un video, y el servidor responde.

A esa parte invisible, la que vive del lado del servidor, la llamamos backend. Durante años se trabajó casi siempre en lenguajes como PHP o Java, hasta que JavaScript, pensado originalmente solo para el navegador, saltó también al servidor.

En las próximas páginas vamos a ver cómo sucedió ese salto y qué protocolos hacen posible que cliente y servidor se entiendan.

BACKSTAGE.JS06

Desarrollo I

// node.js y el javascript del lado del servidor

En 2009 apareció Node.js, un entorno que permite ejecutar JavaScript fuera del navegador, directamente en un servidor.

Gracias a Node, un mismo desarrollador puede escribir el frontend (lo que ve el usuario) y el backend (lo que pasa detrás) en un solo lenguaje.

Junto a Node llegó npm, el gestor de paquetes que permite instalar herramientas y librerías ya hechas en segundos, en vez de programar todo desde cero.

Sobre esa base crecieron frameworks como Express, que simplifican la creación de rutas y respuestas del servidor.

  • Node.js
  • npm
  • Express
  • JSON
  • REST API

Desarrollo II

// los protocolos: el idioma de internet

Para que cliente y servidor se entiendan necesitan hablar el mismo idioma: eso es un protocolo.

  • HTTP
  • HTTPS
  • WebSocket
  • TCP/IP

HTTPS es la versión cifrada de HTTP: misma conversación, pero protegida para que nadie más pueda leerla en el camino.

BACKSTAGE.JS08

Conclusiones

// lo que nos llevamos de esta edición

JavaScript dejó de ser un lenguaje exclusivo del navegador y hoy sostiene servidores completos gracias a Node.js.

Entender los protocolos - HTTP, HTTPS, WebSocket - es entender cómo viajan los datos, no solo cómo se ven en pantalla.

Como futuros ingenieros en sistemas, dominar ambos lados de la web (cliente y servidor) ya no es opcional: es lo que distingue a quien hace páginas de quien construye sistemas.
